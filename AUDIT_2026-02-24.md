**Severity:** HIGH
**File:** `particle/src/core/full_analysis.py`
**Lines:** 2,857

This single file IS the Collider pipeline. It handles:

| `run_full_analysis()` | 1155-2857 | **THE MONOLITH** (1,702 lines) |

### The Problem

`run_full_analysis()` alone is 1,702 lines. It contains the entire 20-stage pipeline
inline: survey, parsing, atom classification, dimension assignment, edge extraction,
scope analysis, data flow, graph metrics, topology, knots, Markov, health scoring,
insights compilation, output generation, and visualization.

You cannot test any stage independently. You cannot swap a stage. You cannot
parallelize stages. Any change risks breaking the entire pipeline.

### The Fix

Extract into focused modules. Keep `full_analysis.py` as thin orchestrator (~200 lines).

| New Module | Functions to Extract | Est. Lines |
|------------|---------------------|------------|
| `disconnection_classifier.py` | `classify_disconnection()` | ~90 |
| `merkle_utils.py` | `calculate_merkle_root()`, `generate_refinery_signature()` | ~130 |
| `js_edge_detection.py` | `detect_js_imports()`, `detect_class_instantiation()` | ~170 |
| `codome_boundary.py` | `create_codome_boundaries()`, `build_file_boundaries()` | ~140 |
| `file_index_builder.py` | `build_file_index()`, `_calculate_theory_completeness()` | ~180 |
| `topology_analysis.py` | `compute_markov_matrix()`, `detect_knots()`, `compute_data_flow()` | ~200 |
| `pipeline_snapshot.py` | `build_pipeline_snapshot()` | ~75 |
| `full_analysis.py` (slim) | `run_pipeline_analysis()`, `run_full_analysis()` | ~300 |

### Validation Criteria

- [ ] Each extracted module has its own test file

## Finding 2: Database Backend Stubs -- Dead Code Pretending to Work

**Severity:** MEDIUM
**Files:**
- `particle/src/core/database/backends/sqlite.py` -- 676 lines, FULLY IMPLEMENTED
- `particle/src/core/database/backends/postgres.py` -- 129 lines, 13 NotImplementedError
- `particle/src/core/database/backends/duckdb.py` -- 169 lines, 16 NotImplementedError

### Evidence

**PostgresBackend -- 13 stub methods:**
`create_run`, `update_run`, `get_run`, `get_runs`, `get_latest_run`,
`insert_nodes`, `get_nodes`, `get_node`, `insert_edges`, `get_edges`,
`get_tracked_files`, `update_tracked_files`, `delete_tracked_files`

**DuckDBBackend -- 16 stub methods:**
Same 13 as Postgres, plus: `export_parquet`, `import_parquet`, `aggregate_metrics`

### The Problem

These stubs create a false impression of multi-backend support. Any code path that
accidentally uses Postgres or DuckDB will crash at runtime with NotImplementedError.
The base class `DatabaseBackend` (477 lines, 30 abstract methods) was designed for
extensibility that never materialized.

### The Fix

**Option A (Recommended):** Delete `postgres.py` and `duckdb.py`. Keep the base class
interface for future use. Add a `TODO: Implement when needed` comment in base.py.

**Option B:** Implement them. Estimated effort: 2-3 sessions per backend.

### Validation Criteria

- [ ] No code imports PostgresBackend or DuckDBBackend (verify before deletion)
- [ ] SQLiteBackend continues to pass all database tests
- [ ] Base class interface preserved for future backends

## Finding 4: Governance Layer Generates Noise

**Severity:** MEDIUM
**Location:** `.agent/tools/` (42 Python scripts, 15K LOC)

### Evidence

| Metric | Value |
|--------|-------|
| Scripts in `.agent/tools/` | 42 |
| Total governance LOC | 14,958 |
| OPPs in inbox | 72 |
| OPPs with `needs_review: true` | 48+ |
| MACRO-001 executions | 73 |
| MACRO-001 successes | 1 (1.4%) |
| MACRO-001 partial failures | 72 (98.6%) |
| Deck cards before cleanup | 15 |
| Deck cards after cleanup | 8 |
| Findings actually fixed | 0 |

### The Pipeline

```
.git/hooks/post-commit
  -> autopilot.py (788 lines)
    -> trigger_engine.py (matches commit patterns to macros)
      -> macro_executor.py (544 lines)
    -> enrichment_orchestrator.py (runs every 24h)
      -> opp_generator.py (generates OPP YAML)
        -> opp_to_deck.py (promotes to cards)
          -> deck_router.py (serves to agents)
```

Six layers of orchestration. Zero completed engineering tasks.

### What Was Fixed This Session

- `opp_to_deck.py`: Eliminated timestamp churn (was regenerating all 15 cards with
  new timestamps on every enrichment run, creating permanent dirty git state)
- Confidence normalization: OPPs stored confidence 0-100, code assumed 0-1 (displayed 9000%)
- Added `_is_promotable()` filter: blocks unenriched imports and completed OPPs from deck
- Purged 10 stale cards (2 completed, 7 unenriched imports, 1 low-confidence)

### Recommendations

1. **Disable MACRO-001** -- 72 failures out of 73 runs. It requires GEMINI_API_KEY and
   Task/Explore agents, neither available from a git hook. It will never succeed in its
   current context.

2. **Simplify the pipeline** -- The 6-layer post-commit chain should be 1 script:
   "If Collider output exists and is newer than last check, generate cards from findings."

3. **Delete or archive** the scripts that produce zero value:
   `lol_sync.py`, `import_legacy_tasks.py`, `priority_matrix.py`, `wire.py`

### Validation Criteria

- [ ] MACRO-001 disabled or fixed to not run from git hooks
- [ ] Post-commit hook completes in <2 seconds
- [ ] No YAML files modified by automation without content changes
