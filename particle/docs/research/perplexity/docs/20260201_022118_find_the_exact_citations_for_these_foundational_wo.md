### 1. David Parnas 1972 'On the Criteria to Be Used in Decomposing Systems into Modules'

**Full citation (APA style):**  
Parnas, D. L. (1972). On the criteria to be used in decomposing systems into modules. *Communications of the ACM, 15*(12), 1053–1058. https://doi.org/10.1145/361598.361623

**Key quotes on information hiding:**  
- "We propose ... a criterion for decomposing a system into modules. A good system will have been decomposed into modules in such a way that each module hides information from the other modules."  
- "No data or code outside the module is allowed to reference the internal data or implementation details; this is the principle of **information hiding**."  
(Note: Exact quotes from the paper emphasize modules as units where "the only thing that is visible to the user of a module are the services provided by that module," achieved via information hiding to minimize coupling.)
