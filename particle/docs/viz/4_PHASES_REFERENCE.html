The 4 Phases

Standard Model of Code - The Kingdoms of Software

Software is organized into 4 fundamental phases, each representing a different aspect of what code does. Like the four elements or the four states of matter, these phases capture the essential nature of code.

DATA is the matter of software - what is stored and manipulated. LOGIC is the forces - what computes and transforms. ORGANIZATION is the structure - how code is grouped and related. EXECUTION is the dynamics - what runs and responds.

Each phase contains 5 families, and each family contains multiple atoms. Together they form the 200-atom taxonomy of the Standard Model.

Phase 1
DATA
"The Matter of Software"
5
Families
28
Atoms
Data is what software stores and manipulates. From the smallest bit to complex data structures, this phase encompasses all forms of information representation. Data atoms are the nouns of programming - the things that exist.
Families
Bits
4 atoms: BitFlag, BitMask, ParityBit, SignBit
Bytes
4 atoms: ByteArray, MagicBytes, PaddingBytes, Buffer
Primitives
10 atoms: Boolean, Integer, Float, String, Null...
Variables
6 atoms: LocalVar, GlobalVar, Parameter, Constant...
Collections
4 atoms: ArrayLiteral, ObjectLiteral, TupleLiteral, SetLiteral
Key Examples
const userId = 42; let items: string[] = []; struct Config { ... }
Key Takeaway
Data phase atoms represent WHAT exists, not what happens. They're passive until Logic acts on them.
Common Confusion
Data vs Organization: A "User class" is Organization (it groups); the "name" field inside is Data.
Typical Levels
L-3 to L1 (Bits to Statements)
⚡
Phase 2
LOGIC
"The Forces of Software"
6
Families
58
Atoms
Logic is what software DOES. Functions, expressions, control flow, and pattern matching are all Logic atoms. This phase contains the verbs of programming - the actions that transform data and make decisions.
Families
Functions
10 atoms: Function, Method, Lambda, Constructor...
Expressions
20 atoms: BinaryExpr, CallExpr, AwaitExpr, SpreadExpr...
Statements
12 atoms: Assignment, ReturnStmt, ThrowStmt...
Control Flow
10 atoms: IfBranch, ForLoop, TryBlock, SwitchCase...
Pattern Matching
6 atoms: MatchPattern, WildcardPattern, GuardPattern...
Key Examples
function calculate(x) { ... } if (condition) { ... } users.map(u => u.name)
Key Takeaway
Logic phase is where computation happens. Pure logic atoms are stateless - same input, same output.
Common Confusion
Logic vs Execution: Logic defines behavior; Execution runs it. A function definition is Logic; calling it is Execution.
Typical Levels
L1 to L3 (Statements to Nodes)
🏗️
Phase 3
ORGANIZATION
"The Structure of Software"
5
Families
52
Atoms
Organization is how software is grouped and structured. Classes, modules, files, and type systems all belong here. This phase contains the containers and boundaries that give shape to code - how pieces relate to each other.
Families
Aggregates
12 atoms: Class, Struct, Enum, Trait, Record...
Modules
8 atoms: Module, Package, ImportStmt, ExportStmt...
Files
6 atoms: SourceFile, Header, ConfigFile, TestFile...
Services/Interfaces
8 atoms: Interface, AbstractClass, Contract, Endpoint...
Type System
18 atoms: TypeRef, GenericParam, UnionType, Lifetime...
Key Examples
class UserService { ... } interface Repository<T> { ... } import { utils } from './lib'
Key Takeaway
Organization is about CONTAINMENT and BOUNDARIES. It determines what can see what and how pieces connect.
Common Confusion
Organization vs Data: A class DEFINITION is Organization; an object INSTANCE is Data. Templates vs instances.
Typical Levels
L3 to L6 (Nodes to Packages)
🚀
Phase 4
EXECUTION
"The Dynamics of Software"
6
Families
62
Atoms
Execution is software at runtime - what actually RUNS. Concurrency, error handling, memory management, I/O, and metaprogramming all live here. This phase deals with the dynamic, temporal aspects of code - what happens when it executes.
Families
Concurrency
12 atoms: Thread, Task, Future, Channel, Mutex...
Error Handling
10 atoms: Exception, Error, Result, Option, Panic...
Memory Management
8 atoms: Allocation, Reference, Borrow, Move, Drop...
I/O Operations
10 atoms: FileRead, NetworkWrite, HttpRequest...
Metaprogramming
12 atoms: MacroDef, Annotation, Reflection, FFI...
Initialization
10 atoms: StaticInit, LazyInit, DefaultInit...
Key Examples
await fetch('/api/users') spawn(|| { process_item() }) try { ... } catch (e) { ... }
Key Takeaway
Execution phase is about TIME and RESOURCES. It deals with what happens, when, and with what resources.
Common Confusion
Execution vs Logic: A "function definition" is Logic; "calling that function" is Execution. Definition vs invocation.
Typical Levels
L3 to L7 (Nodes to Systems)
