LOCUS

The Multi-Dimensional Coordinate System

LOCUS (Latin: "place") is the complete coordinate that locates any code entity in the theory space of the Standard Model. It answers the question: "Where exactly does this piece of code exist in the taxonomy?"

Every atom has exactly one LOCUS - a 5-tuple that captures its hierarchical level, dependency depth, abstraction tier, functional role, and character metric. LOCUS is the theory-space equivalent of a GPS coordinate.

LOCUS differs from file path: a file path is mutable (you can move a file), but LOCUS is determined by the entity's intrinsic properties. If the LOCUS changes, the entity has fundamentally changed.

Key Properties of LOCUS
Uniqueness
Every atom has exactly one LOCUS. No two different atoms can share the same complete coordinate.
Immutability
LOCUS is determined by intrinsic properties, not file location. Moving a file doesn't change its LOCUS.
Identity
If LOCUS changes, the entity is fundamentally different. This solves the Ship of Theseus problem for code.
Composability
LOCUS components are orthogonal - knowing one doesn't determine others. This allows rich classification.
Searchability
LOCUS enables precise queries: "find all L3 R1 Query entities" returns exactly matching functions.
Comparability
LOCUS distance can be computed - entities with similar LOCUS are likely similar in nature.
