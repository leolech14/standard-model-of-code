The 41 Canonical Roles

Standard Model of Code - WHY Dimension

Roles answer the fundamental question: "What is this code FOR?" While atoms describe what code IS (structurally), roles describe its PURPOSE in the system.

The 41 canonical roles emerged from analyzing thousands of real codebases across languages and frameworks. The original 33 roles were extended with 8 SWEBOK-aligned roles to cover software engineering concerns like verification, security, traceability, and project management.

Roles are organized into 11 categories based on their primary function: retrieval, mutation, construction, persistence, coordination, validation, transformation, events, utilities, internal mechanics, and governance.

Query Roles "What retrieves data?" 4 roles
Query CQRS
Data retrieval request - represents intent to read without modification
Examples
GetUserByIdQuery SearchProductsQuery
Key Takeaway
Queries are pure read operations that never mutate state. In CQRS, they're separated from Commands.
Common Confusion
Query vs Finder: Query is the request object; Finder is the function that executes it.
Finder
Search for data based on criteria - the execution logic for queries
Examples
UserFinder.findByEmail() OrderFinder.findPending()
Key Takeaway
Finders encapsulate complex search logic. They return collections or optional results, never single guaranteed items.
Common Confusion
Finder vs Repository: Finders only search; Repositories also handle persistence.
Loader
Load data from storage or external source - bulk or batch retrieval
Examples
ConfigLoader.load() DataLoader.batchLoad(ids)
Key Takeaway
Loaders handle I/O and often implement caching, batching, or lazy loading strategies.
Common Confusion
Loader vs Parser: Loaders fetch raw data; Parsers transform it into structured objects.
Getter
Access a single property or attribute - simple value retrieval
Examples
user.getName() config.getTimeout()
Key Takeaway
Getters should be pure and fast. If computation is involved, consider caching or renaming to compute*.
Common Confusion
Getter vs Finder: Getters access known properties; Finders search for unknown items.
Command Roles "What changes state?" 4 roles
Command CQRS
Intent to modify state - represents a request for change
Examples
CreateUserCommand UpdateOrderStatusCommand
Key Takeaway
Commands express INTENT, not implementation. They're processed by Handlers and may trigger Events.
Common Confusion
Command vs Mutator: Command is the request; Mutator is the function that performs it.
Creator
Create new entities or records - instantiation with business logic
Examples
UserCreator.create(data) OrderCreator.fromCart(cart)
Key Takeaway
Creators often validate input, assign IDs, set defaults, and persist. They're more than simple Factory calls.
Common Confusion
Creator vs Factory: Factories just instantiate; Creators handle the full creation workflow.
Mutator
Modify existing entity or state - in-place updates
Examples
user.updateEmail(newEmail) order.addItem(item)
Key Takeaway
Mutators change internal state. They should validate invariants and may emit domain events.
Common Confusion
Mutator vs Setter: Mutators enforce business rules; Setters are raw property assignments.
Destroyer
Delete or remove entities - cleanup and removal logic
Examples
UserDestroyer.delete(userId) cache.invalidate(key)
Key Takeaway
Destroyers handle cascading deletes, soft-delete logic, and cleanup of related resources.
Common Confusion
Destroyer vs Lifecycle: Destroyers explicitly delete; Lifecycle hooks handle automatic cleanup.
Factory Roles "What builds objects?" 2 roles
Factory
Create and configure new instances - encapsulated instantiation
Examples
ConnectionFactory.create(config) LoggerFactory.getLogger(name)
Key Takeaway
Factories hide construction complexity and enable dependency injection. They return ready-to-use objects.
Common Confusion
Factory vs Builder: Factories create in one step; Builders construct incrementally.
Builder
Construct complex objects step by step - fluent API pattern
Examples
QueryBuilder.where().orderBy().limit() HttpRequest.builder().url().headers().build()
Key Takeaway
Builders enable readable construction of objects with many optional parameters. Always end with build().
Common Confusion
Builder vs Mapper: Builders create new objects; Mappers transform existing ones.
Storage Roles "What persists data?" 3 roles
Repository DDD
Abstract data persistence operations - collection-like interface for aggregates
Examples
UserRepository.save(user) OrderRepository.findById(id)
Key Takeaway
Repositories act as in-memory collections that happen to persist. They hide database details from domain logic.
Common Confusion
Repository vs DAO: Repositories work with domain objects; DAOs work with database rows.
Store
Manage application state - centralized state container
Examples
Redux store.getState() Vuex store.commit(mutation)
Key Takeaway
Stores are the single source of truth for UI state. Changes flow through actions/mutations only.
Common Confusion
Store vs Cache: Stores manage app state; Caches optimize data access.
Cache
Temporarily store data for performance - avoid repeated computation or I/O
Examples
Redis.get(key) memoizedFunction(args)
Key Takeaway
Caches are always optional - the system must work without them. They have TTL and invalidation strategies.
Common Confusion
Cache vs Store: Caches are ephemeral and can be cleared; Stores hold authoritative state.
Orchestration Roles "What coordinates work?" 4 roles
Service
Coordinate business operations - stateless business logic
Examples
PaymentService.processPayment() NotificationService.send()
Key Takeaway
Services are stateless and coordinate between entities. They contain logic that doesn't belong to any single entity.
Common Confusion
Service vs Manager: Services execute business logic; Managers handle resource lifecycle.
Controller
Handle external requests - entry point for outside world
Examples
UserController.handleCreate(req) @GetMapping("/users")
Key Takeaway
Controllers are thin - they translate HTTP to domain calls. Business logic belongs in Services.
Common Confusion
Controller vs Handler: Controllers handle HTTP; Handlers can process any event type.
Manager
Manage resources or components - lifecycle and pooling
Examples
ConnectionManager.getConnection() ThreadPoolManager.submit(task)
Key Takeaway
Managers own resources - they allocate, track, and release. They're often singletons.
Common Confusion
Manager vs Orchestrator: Managers handle resources; Orchestrators coordinate multi-step workflows.
Orchestrator
Coordinate complex workflows - multi-step process coordination
Examples
CheckoutOrchestrator.execute() SagaOrchestrator.runSaga()
Key Takeaway
Orchestrators know the workflow sequence. They call multiple services and handle compensation on failure.
Common Confusion
Orchestrator vs Service: Orchestrators manage flow; Services contain actual business logic.
Validation Roles "What checks correctness?" 5 roles
Validator
Verify data meets constraints - input validation
Examples
EmailValidator.isValid(email) @Valid CreateUserDto
Key Takeaway
Validators check input at system boundaries. They return boolean or throw with detailed error messages.
Common Confusion
Validator vs Guard: Validators check data format; Guards check authorization/permissions.
Verifier SWEBOK
Confirm build correctness - "are we building it right?"
Examples
SignatureVerifier.verify(token) BuildVerifier.checkArtifacts()
Key Takeaway
Verifiers ensure the output matches specification. They're part of quality assurance, not input validation.
Common Confusion
Verifier vs Validator: Verifiers check outputs/results; Validators check inputs/data.
Guard
Protect access or enforce conditions - authorization checks
Examples
AuthGuard.canActivate(context) RoleGuard.checkPermission(user)
Key Takeaway
Guards are gatekeepers. They run before handlers and can short-circuit execution with 401/403.
Common Confusion
Guard vs Middleware: Guards check conditions; Middleware transforms request/response.
Asserter
Assert truth of conditions - invariant enforcement
Examples
assert(balance >= 0) expect(result).toBe(expected)
Key Takeaway
Asserters crash loudly on failure. They're for programmer errors, not user input errors.
Common Confusion
Asserter vs Validator: Asserters throw on failure; Validators return validation results.
SecurityValidator SWEBOK
Security-specific validation - threat prevention
Examples
XSSValidator.sanitize(input) SQLInjectionValidator.check(query)
Key Takeaway
SecurityValidators specifically prevent security vulnerabilities - XSS, SQLi, CSRF, etc.
Common Confusion
SecurityValidator vs Guard: SecurityValidators sanitize data; Guards check identity/permissions.
Transform Roles "What converts data?" 4 roles
Transformer
Convert data format - general-purpose transformation
Examples
ImageTransformer.resize() DataTransformer.normalize()
Key Takeaway
Transformers take input A and produce output B. They're pure functions when possible.
Common Confusion
Transformer vs Mapper: Transformers change data; Mappers change types/structure.
Mapper
Map between types - structural conversion
Examples
UserMapper.toDTO(entity) OrderMapper.fromRequest(req)
Key Takeaway
Mappers convert between layer boundaries - Entity to DTO, Request to Command, etc.
Common Confusion
Mapper vs Serializer: Mappers convert between objects; Serializers convert to/from bytes.
Serializer
Convert objects to output format - object to bytes/string
Examples
JSON.stringify(obj) ProtobufSerializer.encode()
Key Takeaway
Serializers produce wire format for storage or transmission. Paired with Parsers for round-trip.
Common Confusion
Serializer vs Formatter: Serializers produce machine-readable; Formatters produce human-readable.
Parser
Convert input format to structured data - bytes/string to object
Examples
JSON.parse(str) XMLParser.parse(document)
Key Takeaway
Parsers handle syntax and produce AST/objects. They should fail fast on invalid input.
Common Confusion
Parser vs Loader: Parsers transform format; Loaders fetch from sources.
Event Roles "What reacts to events?" 4 roles
Handler
Process events or requests - active event processing
Examples
OrderCreatedHandler.handle(event) WebSocketHandler.onMessage()
Key Takeaway
Handlers DO something in response to events. They contain the reaction logic.
Common Confusion
Handler vs Listener: Handlers process; Listeners just observe and delegate.
Listener
Passive event observation - subscribe and delegate
Examples
document.addEventListener('click') @EventListener(UserCreated.class)
Key Takeaway
Listeners are the subscription mechanism. They receive events and route to appropriate handlers.
Common Confusion
Listener vs Subscriber: Listeners are synchronous; Subscribers often imply async/message queues.
Subscriber
Subscribe to event streams - async message consumption
Examples
observable.subscribe(observer) kafka.subscribe(topic, handler)
Key Takeaway
Subscribers consume from streams/queues. They handle backpressure and acknowledgment.
Common Confusion
Subscriber vs Handler: Subscribers manage the subscription; Handlers process individual messages.
Emitter
Publish events - event production
Examples
eventEmitter.emit('userCreated', data) domainEvents.publish(event)
Key Takeaway
Emitters decouple producers from consumers. They don't know who's listening.
Common Confusion
Emitter vs Publisher: Often synonymous; Publisher sometimes implies external message brokers.
Utility Roles "What helps?" 5 roles
Utility
General stateless helper - pure utility functions
Examples
StringUtils.capitalize(str) ArrayUtils.chunk(arr, size)
Key Takeaway
Utilities are pure, stateless, and reusable. They belong in shared modules.
Common Confusion
Utility vs Helper: Utilities are generic; Helpers are context-specific.
Formatter
Format output for display - human-readable presentation
Examples
DateFormatter.format(date, 'YYYY-MM-DD') CurrencyFormatter.format(amount)
Key Takeaway
Formatters are locale-aware and produce human-readable strings, not machine-readable data.
Common Confusion
Formatter vs Serializer: Formatters are for humans; Serializers are for machines.
Helper
Specific assistive logic - context-aware assistance
Examples
AuthHelper.getCurrentUser() TestHelper.createMockUser()
Key Takeaway
Helpers know about their context. They simplify repetitive patterns within a specific domain.
Common Confusion
Helper vs Service: Helpers are simple and focused; Services coordinate business logic.
StatisticalAnalyzer SWEBOK
Data analysis - metrics and statistical computation
Examples
MetricsAnalyzer.calculateP95(data) QualityAnalyzer.computeDefectDensity()
Key Takeaway
StatisticalAnalyzers compute metrics for quality assurance - coverage, complexity, defect rates.
Common Confusion
StatisticalAnalyzer vs Measurer: Analyzers compute derived metrics; Measurers collect raw data.
Tracer SWEBOK
History and provenance tracking - traceability
Examples
RequestTracer.trace(spanId) RequirementTracer.linkToCode()
Key Takeaway
Tracers maintain the audit trail - who changed what, when, and why. Essential for compliance.
Common Confusion
Tracer vs Logger: Tracers track relationships/provenance; Loggers record events.
Internal Roles "What manages internals?" 2 roles
Internal
Private implementation detail - encapsulated logic
Examples
_privateHelper() __internalState
Key Takeaway
Internal code is not part of public API. It can change without notice.
Common Confusion
Internal vs Helper: Internal is truly private; Helpers may be shared within a module.
Lifecycle
Manage object/process lifecycle - init, cleanup, hooks
Examples
componentDidMount() / ngOnInit() @PreDestroy cleanup()
Key Takeaway
Lifecycle hooks run at specific phases - creation, update, destruction. They're framework-managed.
Common Confusion
Lifecycle vs Destroyer: Lifecycle is automatic; Destroyer is explicit deletion.
Management Roles "What governs?" 4 roles
ProjectManager SWEBOK
Resource and schedule coordination - project governance
Examples
TaskScheduler.schedule(task) ResourceAllocator.assign()
Key Takeaway
ProjectManager code handles scheduling, resource allocation, and coordination - the meta-level of work.
Common Confusion
ProjectManager vs Orchestrator: ProjectManager handles resources; Orchestrator handles workflows.
RiskManager SWEBOK
Risk identification and mitigation - failure prevention
Examples
CircuitBreaker.execute(fn) RateLimiter.checkLimit()
Key Takeaway
RiskManagers prevent cascading failures - circuit breakers, rate limiters, bulkheads, retries.
Common Confusion
RiskManager vs Guard: RiskManagers prevent system failures; Guards prevent unauthorized access.
Certifier SWEBOK
Professional/product certification - quality attestation
Examples
CertificateAuthority.issue() ComplianceChecker.attest()
Key Takeaway
Certifiers formally attest that something meets standards - SSL certs, compliance reports, signatures.
Common Confusion
Certifier vs Verifier: Certifiers issue formal attestation; Verifiers just check correctness.
Measurer SWEBOK
Metrics collection - raw measurement
Examples
PerformanceMonitor.measure(fn) CoverageCollector.collect()
Key Takeaway
Measurers collect raw data points - timing, counts, sizes. They feed into analyzers and dashboards.
Common Confusion
Measurer vs StatisticalAnalyzer: Measurers collect raw data; Analyzers compute derived metrics.
