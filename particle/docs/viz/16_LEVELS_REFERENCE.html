The 16-Level Scale

Standard Model of Code - Hierarchical Position (λ)

Levels answer the question: "How big is this entity?" The scale ranges from L-3 (individual bits) to L12 (the entire software industry), spanning 16 orders of magnitude.

Each level represents a natural boundary in software organization. Code at different levels has different concerns, lifespans, and change frequencies. The Operational Zone (L3-L7) is where Collider focuses its classification - from functions to systems.

The scale is divided into 4 zones: Syntactic (machine-level), Semantic (meaning-level), Systemic (architecture-level), and Cosmological (ecosystem-level).

Cosmological Zone L8 - L12
L12 UNIVERSE
The entire software industry - all code ever written
Examples
The global software ecosystem All open source + proprietary code
Key Takeaway
L12 is theoretical - it represents the totality of software. Useful for thinking about universal patterns.
Common Confusion
Universe vs Domain: Universe is ALL software; Domain is a specific industry vertical.
L11 DOMAIN
Industry vertical or problem domain - healthcare, finance, gaming
Examples
Healthcare software industry FinTech ecosystem
Key Takeaway
Domains have shared vocabularies, regulations, and patterns. DDD "bounded contexts" live here.
Common Confusion
Domain (L11) vs domain logic: L11 is industry-wide; domain logic is your app's business rules.
L10 ORGANIZATION
Company, foundation, or team - the human structure that owns code
Examples
Google, Microsoft, Apache Foundation Your company's entire codebase
Key Takeaway
Organizations determine conventions, tooling, and governance. Conway's Law operates here.
Common Confusion
Organization vs Platform: Organizations are human structures; Platforms are technical infrastructure.
L9 PLATFORM
Infrastructure platform - AWS, Kubernetes, Browser, OS
Examples
AWS Lambda, Google Cloud Run Kubernetes, Docker Swarm
Key Takeaway
Platforms provide primitives and constraints. Your code's capabilities are bounded by L9.
Common Confusion
Platform vs Ecosystem: Platforms run code; Ecosystems distribute code.
L8 ECOSYSTEM
Package registry and language community - npm, PyPI, crates.io
Examples
npm (2M+ packages) PyPI, Maven Central, crates.io
Key Takeaway
Ecosystems determine what libraries exist. Your dependencies are drawn from L8.
Common Confusion
Ecosystem vs System: Ecosystem is the shared registry; System is your deployed app.
Systemic Zone L4 - L7 (Operational)
L7 SYSTEM
Deployable application unit - microservice, monolith, CLI tool
Examples
user-service, payment-gateway collider CLI, VS Code
Key Takeaway
L7 is the deployment boundary. It has its own lifecycle, versioning, and API surface.
Common Confusion
System vs Package: Systems are deployed; Packages are imported.
L6 PACKAGE
Distributable module - npm package, Python module, Rust crate
Examples
lodash, requests, serde src/utils/, lib/core/
Key Takeaway
L6 is the sharing boundary. Packages have public APIs and private internals.
Common Confusion
Package vs Directory: Not all directories are packages; packages have exports.
L5 FILE
Source file - the fundamental unit of code organization
Examples
user_service.py, OrderController.ts main.go, lib.rs
Key Takeaway
L5 is the git/editor boundary. Files are what you open, diff, and commit.
Common Confusion
File vs Module: In Python/Rust they're the same; in JS/TS they differ.
L4 CONTAINER
Type container - class, struct, interface, enum
Examples
class UserService { } struct Config { }, enum Status { }
Key Takeaway
L4 is the encapsulation boundary. Containers group related functions and data.
Common Confusion
Container vs Node: Containers hold multiple nodes; a Node is a single function.
Semantic Zone L1 - L3
L3 NODE
Callable unit - function, method, property, hook
Examples
function getUserById(id) { } def process_payment(self): ...
Key Takeaway
L3 is the unit of behavior. Functions are tested, documented, and called.
Common Confusion
Node vs Block: Nodes have names and signatures; Blocks are anonymous control structures.
L2 BLOCK
Control structure - if/else, loops, try/catch
Examples
if (condition) { ... } for item in collection: ...
Key Takeaway
L2 is the control flow boundary. Blocks create scope and branch execution.
Common Confusion
Block vs Statement: Blocks contain statements; Statements are single lines.
L1 STATEMENT
Single executable line - assignment, return, import
Examples
const x = computeValue(); return result;
Key Takeaway
L1 is the execution unit. Statements are what debuggers step through.
Common Confusion
Statement vs Expression: Statements do things; Expressions return values.
Syntactic Zone L-3 - L0
L0 TOKEN
Lexical unit - identifier, keyword, operator, literal
Examples
function, const, =>, 42, "hello" userId, async, await
Key Takeaway
L0 is the lexer output. Tokens are the atoms of syntax.
Common Confusion
Token vs Character: Tokens have meaning; Characters are just bytes.
L-1 CHARACTER
Single character - the visible representation
Examples
'a', 'b', '{', '}' Unicode: '字', '🚀'
Key Takeaway
L-1 is the display unit. Characters are what you see in editors.
Common Confusion
Character vs Byte: In UTF-8, one character can be multiple bytes.
L-2 BYTE
8-bit value - the storage unit
Examples
0x61 ('a'), 0x7B ('{') UTF-8 encoding sequences
Key Takeaway
L-2 is the I/O unit. Files are byte streams.
Common Confusion
Byte vs Bit: A byte is 8 bits. Network bandwidth is in bits; storage is in bytes.
L-3 BIT
Binary digit - the fundamental unit of information
Examples
0, 1 BitFlag, BitMask
Key Takeaway
L-3 is the hardware boundary. Everything ultimately reduces to bits.
Common Confusion
Bit vs Boolean: Booleans have semantics (true/false); bits are just 0/1.
