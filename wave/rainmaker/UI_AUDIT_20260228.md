## P0 -- Silent Failures (Fix Immediately)

### 1. Undefined tokens in finance.css

```css
/* finance.css ~L249 -- spinner border renders as nothing */
border-top: 2px solid var(--accent);     /* --accent is UNDEFINED */

/* finance.css ~L342 -- input focus state broken */
border-color: var(--primary);            /* --primary is UNDEFINED */
```

**Fix:** Add to `tokens.css` or replace inline:
```css
border-top: 2px solid var(--feedback-info);
border-color: var(--input-border-focus);
```

### 2. .btn:hover is a no-op

```css
/* style.css -- hover identical to default */
.btn              { background: var(--surface-hover); }
.btn:hover        { background: var(--surface-hover); }  /* same value */
```

**Fix:** Use different tokens for default vs hover:
```css
.btn       { background: var(--interactive-bg); }
.btn:hover { background: var(--interactive-bg-hover); }
```

## P1 -- Design System Integrity

### 3. Legacy vs new token namespace

`--text-bright` (style.css legacy) and `--text-primary` (tokens.css) both resolve to `#e0e0e0`. Situation Room uses `--text-primary`, everything else uses `--text-bright`. A future change to one silently breaks the other.

**Fix:** Audit all `--text-bright` uses, replace with `--text-primary`, remove legacy alias.

### 4. Hardcoded colors in automations.css

```css
.hero-ok .auto-hero-val   { color: #10b981; }  /* green */
.hero-warn .auto-hero-val { color: #f59e0b; }  /* amber */
.hero-error .auto-hero-val{ color: #ef4444; }  /* red */
```

These bypass the B&W token system entirely. The automations page looks like a different app.

**Decision:** Either activate real colors system-wide in tokens.css:
```css
--p-green: #10b981;   /* currently #a0a0a0 */
--p-yellow: #f59e0b;  /* currently #888888 */
--p-red: #ef4444;     /* currently #777777 */
```
...or remove these overrides and use the monochrome scale.

### 5. Situation Room colors need tokens

```css
.text-red        { color: #f87171; }           /* hardcoded */
.glow-red        { box-shadow: ... #b91c1c; }  /* hardcoded */
.sr-alert-pulse  { background: #ef4444; }       /* hardcoded */
```

These are justified (alerts need color). But they should participate in the token cascade:
```css
/* tokens.css T2: */
--alert-critical: #ef4444;
--alert-critical-text: #fca5a5;
--alert-critical-glow: rgba(185, 28, 28, 0.3);
--alert-warn: #eab308;
--alert-warn-text: #fef08a;
```

## P2 -- Component & Layout

### 6. Cloud widget drawer doesn't animate

```css
/* width: auto cannot transition in CSS */
.rm-drawer { width: 0; transition: all 0.4s; }
.active .rm-drawer { width: auto; }  /* pops, doesn't slide */
```

**Fix:** Use `max-width` instead:
```css
.rm-drawer { max-width: 0; overflow: hidden; }
.active .rm-drawer { max-width: 400px; }
```

### 7. Toast/widget z-index conflict on mobile

Toast: `z-index: 100`, `bottom: 12px`.
Widget: `z-index: 9999`, `bottom: 20px`.
Widget occludes toast on mobile.

**Fix:** Raise toast to `z-index: 10000`.

### 8. Card padding inconsistency

`.card` (style.css): `padding: 14px` (raw)
`.rc-card` (components.css): `padding: var(--card-padding)` = 16px

Both are used on same pages. **Fix:** Standardize on `var(--card-padding)`.

### 9. Five table implementations

`.pos-table`, `.data-table`, `.rc-table`, `.tp-table`, `.ops-svc-table` all do the same thing.

**Fix:** Consolidate to `rc-table` as canonical. Add modifier classes for page-specific spacing.

## P3 -- Power User UX

### 10. No global command palette

28 pages, 6 pillars, dropdown-only navigation. Keyboard users are slow.

**Fix:** Add `Cmd+K` fuzzy search over existing nav link labels. ~50 LOC JS.

### 11. Sub-nav scroll has no overflow indicator

Horizontal sub-nav scrolls but no visual cue that more items exist.

**Fix:** Right-edge gradient fade:
```css
.sub-nav-wrapper::after {
    content: '';
    position: absolute; right: 0; top: 0; bottom: 0;
    width: 32px;
    background: linear-gradient(to right, transparent, var(--surface));
    pointer-events: none;
}
```

### 12. Status dot has no tooltip

Header status dot communicates health but no explanation of what's degraded.

**Fix:** Add `title` attribute or hover tooltip linking to command center.

### 13. Finance page glassmorphism breaks the system

`.glass-panel`, `.text-gradient`, `hover: translateY(-2px)` are a completely different visual language from the rest of the system. Undocumented inconsistency.

**Decision:** Align to monochrome or document as intentional zone.

### 14. .mr-value uses raw 48px

Margin ratio gauge heading is `font-size: 48px` -- outside the type scale (max is `--p-text-5xl: 32px`).

**Fix:** Add `--p-text-6xl: 48px` to T1 primitives.

## WCAG Contrast Reference

| Token | Hex | On #000 | WCAG AA (4.5:1) |
|-------|-----|---------|-----------------|
| `--text-primary` | #e0e0e0 | 16.03:1 | PASS AAA |
| `--text-secondary` | #a0a0a0 | 9.97:1 | PASS AA |
| `--text-dim` | #666666 | 3.83:1 | FAIL |
| `--text-muted` | #555555 | 2.96:1 | FAIL |
| `--text-disabled` | #333333 | 1.73:1 | N/A (disabled) |

`--text-muted` is used for card titles, gauge labels, tab labels, table headers -- content that carries meaning. Accepted trade-off for personal tool, but worth explicit acknowledgment.

## What's Award-Worthy

1. **Login page.** Conceptually pure. 20px line as the only accent. Better than any logo.
2. **Dropdown hover bridge.** `::before` invisible zone preventing diagonal-mouse dismissal. Most dashboards get this wrong.
3. **3-tier token cascade.** T1 -> T2 -> T3 with light mode overriding only T1. Genuinely sophisticated.
4. **Cloud widget lightning.** 8-second standby flicker at 95/95.5/95.8% keyframe timing. Character work.
5. **Trade page layout.** `clamp(180px, 18vw, 230px)` sidebar + resizable divider. Professional-grade terminal.
6. **View Transition API.** CSS `@view-transition` with header persistence and `@supports` fallback. Cutting-edge.
