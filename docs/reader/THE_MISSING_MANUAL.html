The Missing Manual

The Missing Manual: Don’t Panic

“The only way to understanding is through the collapse of the wave function.”

You Are Overwhelmed

If you are reading this, you are likely overwhelmed by the 13GB of data, the 16 levels of theory, and the sheer density of the Standard Model of Code.

This is normal. You are looking at a “Grand Unified Theory” of software, not just a linter. It is designed to be dense because it is modeling complexity itself.

The Top 3 Things You Need to Know

1. The Project is a Collider

Just as CERN smashes particles to find bosons, this project smashes Source Code to find Atoms of Logic. * Input: A git repository (like this one). * Process: The collider engine (standard-model-of-code/src/core/collider). * Output: A graph of 100,000+ nodes connecting every function, class, and variable.

One Command to Rule Them All:

./collider full /path/to/repo --output /tmp/data

2. The Two Hemispheres

The project is split into two biological halves:

  • The Body (standard-model-of-code): The physics engine. It is deterministic, rust-like (though written in Python/JS), and rigourous. It handles the syntax and structure.
  • The Brain (context-management): The intelligence layer. It is probabilistic, AI-driven, and fluid. It handles the semantics and meaning.

If you want to change how we parse code, go to the Body. If you want to change how we understand code, go to the Brain.

3. The Map is the Territory

Documentation is not a separate activity here. The Reader you are looking at is generated from the code structure itself. * L0 - L3: The Theory Stack (docs/theory/). * Governance: How we work (governance/). * The Atlas: The visual map of the system.

Where to Start?

  1. Read the Visual Guide: Visual Guide (in Governance).
  2. Explore the Reader: Use the sidebar to click through the standard-model-of-code definitions.
  3. Run a Scan: Try running the collider on a small repo to see the physics in action.

Troubleshooting

  • “It’s too much”: Focus on Level 1 (Definitions). Ignore the Laws and Applications for now.
  • “Where is x?”: Use the socratic tool (AI) or grep. The file names are strictly strictly semantic (e.g., CORPUS_INVENTORY.py does exactly what it says).
